Early Protocols

What made the internet today comprises of protocols that helped create the main function of the internet: sending and receiving information over the network. Some of these protocols are:

NCP-Network Control Program

This program provides the middle layer of protocol stacks for the ARPAnet, where it acted as a transport layer. According to Wikipedia, this protocol provides connections and flow control between processes in the ARPAnet. It is the predecessor of TCP.

TCP/IP –Transmission Control Protocol -Internet Protocol

These 2 are different but are linked together because they are complementary. It was very vital that it was even said that we have the internet today because of these. According to Margaret Rouse from searchnetworking.techtarget.com, TCP/IP is a two layer program. The first one is for assembling of files into smaller packets to be sent over the internet and received by a TCP layer that will reassemble the packets similar to the original. The Internet Protocol, the lower layer, is responsible for the address of each packet so it will arrive to the right destination.

This protocol uses the client-server model of communication (point to point) where a client or a user requests for the information and the server supplies the service or information requested.

FTP-File Transfer Protocol

This is a network protocol that is used for transferring or exchanging of computer files from one host to another. Like the TCP/IP, it is also based on a client-server communication, where logging in to authenticate the user or in an anonymous mode where the former is not needed can be used if the server allows it. Also, FTP works in a similar way with HTTP and SMTP; the former for exchanging of files and the latter for transferring e-mail across the internet.

NVP- Network Voice Protocol (Dec. 1973)

During its time, it was used to send speech between sites on the ARPAnet. It is composed of two parts: control and data transport protocol. According to Wikipedia, “Control protocols included relatively rudimentary telephony features such as indicating who wants to talk to whom; ring tones; negotiation of voice encoding; and call termination. Data messages contained encoded speech. For each encoding scheme (vocoder) a frame was defined as a packet containing the negotiated transmission interval of a number of digitized voice samples.”
